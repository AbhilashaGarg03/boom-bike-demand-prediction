# Contributing to Boom Bike Demand Prediction

Thank you for your interest in contributing to this project! We welcome contributions from the community.

## How to Contribute

### Reporting Issues
- Check existing issues before creating a new one
- Provide a clear description of the issue
- Include relevant data, screenshots, or error messages
- Specify your environment (Python version, OS, etc.)

### Submitting Changes

1. **Fork the repository**
   ```bash
   git clone https://github.com/YOUR_USERNAME/boom-bike-demand-prediction.git
   ```

2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Write clear, descriptive commit messages
   - Add comments for complex logic
   - Update documentation as needed

4. **Install development dependencies**
   ```bash
   pip install -r requirements.txt
   ```

5. **Test your changes**
   - Run notebooks to verify output
   - Check for any data inconsistencies
   - Validate model performance

6. **Push to your fork and submit a Pull Request**
   ```bash
   git push origin feature/your-feature-name
   ```

## Code Style Guidelines

- Follow PEP 8 style guide for Python
- Use descriptive variable names
- Add docstrings to functions
- Include comments for complex sections

## Areas for Contribution

- Improving model performance
- Adding new features or datasets
- Enhancing documentation
- Fixing bugs
- Adding test cases

## Pull Request Process

1. Ensure your code works without errors
2. Update README.md if needed
3. Provide a clear description of changes
4. Reference any related issues

## Questions?

Feel free to open an issue with the `question` label.

Thank you for contributing! 🙏
